# MCP Simple — Math Tools Server

A minimal [FastMCP](https://github.com/jlowin/fastmcp) server exposing two tools (`add`, `subtract`) over HTTP, with a matching Python client.

---

## How MCP Works

**Model Context Protocol (MCP)** is an open protocol that lets AI models (or any client) discover and invoke tools hosted on a server over a standard HTTP/JSON interface.

```
┌──────────────┐        JSON-RPC 2.0        ┌──────────────────┐
│  MCP Client  │  ──────────────────────►   │   MCP Server     │
│ (client.py)  │  ◄──────────────────────   │  (tools.py)      │
└──────────────┘        HTTP/SSE             └──────────────────┘
```

### Key concepts

| Concept | Description |
|---|---|
| **Tool** | A named function the server exposes (e.g. `add`, `subtract`) |
| **Tool schema** | JSON Schema describing each tool's inputs/outputs, auto-generated by FastMCP |
| **Tool call** | A client request to invoke a specific tool with arguments |
| **Transport** | HTTP + Server-Sent Events (SSE) or WebSockets; here we use HTTP |

### Tool calling flow

1. **Discovery** — client sends `tools/list` to learn what tools are available and their input schemas.
2. **Invocation** — client sends `tools/call` with the tool name and a JSON arguments object.
3. **Response** — server executes the function and returns the result as a JSON payload.

---

## Why `Accept: application/json, text/event-stream` Is Required

**You cannot omit `text/event-stream`** — this is mandated by the [MCP Streamable HTTP transport spec](https://modelcontextprotocol.io/docs/concepts/transports):

> *"Clients MUST include `Accept: application/json, text/event-stream` on all requests to the MCP endpoint."*

### Why both are needed

The MCP protocol allows the server to respond in **either** format depending on the message:

- Simple results → `application/json`
- Streamed / long-running responses → `text/event-stream` (SSE)

Because the **server decides at runtime** which format to use, the client must declare upfront that it can handle both. Omitting `text/event-stream` causes the server to reject the request with error `-32600 Not Acceptable`.

### Transport comparison

| Transport | `text/event-stream` required? |
|---|---|
| **Streamable HTTP** (FastMCP default) | **Yes, mandatory** |
| **stdio** | N/A — no HTTP involved |
| Raw HTTP without MCP SDK | You'd need to implement the protocol manually — not advisable |

> The FastMCP `Client` sets both headers automatically, which is why `client.py` works without manually specifying them.

```bash
python client.py
```

## Why `client.py` Doesn't Need `initialize` (But Postman Does)

`client.py` **does** run `initialize` — it's just invisible. The `async with client:` context manager automatically:

1. Sends `initialize` to the server
2. Receives and stores the `Mcp-Session-Id` from the response header
3. Attaches that session ID to every subsequent request
4. Tears down the session on exit

```python
async with client:                        # ← initialize happens here, session ID stored
    result = await client.call_tool(...)  # ← Mcp-Session-Id sent automatically
                                          # ← session closed on exit
```

Postman sends raw JSON-RPC with no SDK managing state, so you replicate those same steps manually. The flow is identical — just explicit.

## Trying with Postman (manual equivalent of `client.py`)

Base URL: `http://localhost:8000`

> **Every POST to `/mcp` needs these two headers:**
> - `Content-Type: application/json`
> - `Accept: application/json, text/event-stream`

---

### Step 1 — Initialize session (do this first, every time)

```
POST http://localhost:8000/mcp
Content-Type: application/json
Accept: application/json, text/event-stream

{
  "jsonrpc": "2.0",
  "id": 0,
  "method": "initialize",
  "params": {
    "protocolVersion": "2024-11-05",
    "capabilities": {},
    "clientInfo": { "name": "postman", "version": "1.0" }
  }
}
```

In the **response headers**, copy the value of `Mcp-Session-Id` (e.g. `abc123`).  
Add it as a header on all subsequent requests:
```
Mcp-Session-Id: <value from response>
```

---

### Step 2 — Discover available tools

```
POST http://localhost:8000/mcp
Content-Type: application/json
Accept: application/json, text/event-stream
Mcp-Session-Id: <your-session-id>

{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/list",
  "params": {}
}
```

---

### Step 3 — Call `add`

```
POST http://localhost:8000/mcp
Content-Type: application/json
Accept: application/json, text/event-stream
Mcp-Session-Id: <your-session-id>

{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "add",
    "arguments": { "a": 10, "b": 5 }
  }
}
```

**Expected response:**
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "content": [{ "type": "text", "text": "15.0" }]
  }
}
```

## cURL equivalents

```bash
# Step 1 — Initialize and capture session ID
SESSION_ID=$(curl -s -D - -X POST http://localhost:8000/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","id":0,"method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"curl","version":"1.0"}}}' \
  | grep -i "mcp-session-id" | awk '{print $2}' | tr -d '\r')

echo "Session ID: $SESSION_ID"

# Step 2 — List tools
curl -X POST http://localhost:8000/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -H "Mcp-Session-Id: $SESSION_ID" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'

# Step 3 — Call add
curl -X POST http://localhost:8000/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -H "Mcp-Session-Id: $SESSION_ID" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"add","arguments":{"a":10,"b":5}}}'

# Step 4 — Call subtract
curl -X POST http://localhost:8000/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -H "Mcp-Session-Id: $SESSION_ID" \
  -d '{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"subtract","arguments":{"a":10,"b":5}}}'
```
